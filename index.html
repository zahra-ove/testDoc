1.2.configuration #

For configuring Project, in public_html directory open index.php file.
There are three lines of code that must be edited properly based on your need.

define('cor', dirname(__DIR__, 2) . '/__frm.v4/core/');
define('app', dirname(__DIR__) . '/application/');
define('www', __DIR__ . '/');

If in installation step, the tree directory is set exactly like Pic1.2, then there is no need to change these three lines of code, otherwise these codes should be changed based on public_html path.

  • The define('cor', dirname(__DIR__, 2) . '/__frm.v4/core/');
    determines core directory path that is placed in __frm.v4 framework which is downloaded in step 1.

  • The define('app', dirname(__DIR__) . '/application/');
    determines application directory path.

  • The define('www', __DIR__ . '/');
    determines the public_html path.

And finally change this line based on your environment:
define('dev', 1);
defines current environment. in development stage leave it as 1, otherwise change it to 0. changing dev to 0, prevents from error displaying.

1.3.index file #

There is other defines and configs in index.php file that is not important for configuration, but we study them here for more information.

  • define('ajax', strtolower('XMLHttpRequest') === @strtolower($_SERVER['HTTP_X_REQUESTED_WITH'])); determines type of request, if request type is ajax then this constant equals to 1, otherwise it is set to 0.

  • define('log', app . '/logs/'); defines error logs path in the application.

  • ini_set('log_errors', 1);
    Determines whether script error messages should be logged to the server's error log or error_log. This option is thus server-specific[ref].

  • ini_set('error_log', log . PHP_OS . '-' . date('Y-m-d') . '.log');
    Name of the file where script errors should be logged. The file should be writable by the web server's user[ref].

  • ini_set('display_errors', dev ? 'On' : 'off');
    This determines whether errors should be printed to the screen as part of the output or if they should be hidden from the user[ref].

  • error_reporting(dev ? E_ALL : 0);
    In development mode, displays all errors and in production mode does not display any error to the screen.

in the following we have,

  • date_default_timezone_set('UTC');
    Sets the default timezone to Universal Time Coordinated (UTC). It refers to time on the zero or Greenwich meridian.

  • In this code If any of cor, app or www directories are not defined, then throw an exception.

    if (!(is_dir(cor) and is_dir(app) and is_dir(www)))
throw new Exception(json_encode([
    'cor' => (int)is_dir(cor),
    'app' => (int)is_dir(app),
    'www' => (int)is_dir(www)
]));

  • If autoload.php or other.php files are not defined, then an exception is thrown. otherwise these two files are required in the following next lines. 
    if (!is_file(cor . 'functions/autoload.php') OR !is_file(cor . 'functions/other.php'))
throw new Exception(json_encode([
    'cor._autoload' => is_file(cor . 'functions/autoload.php'),
    'cor.other' => is_file(cor . 'functions/other.php')
]));

After that the application will be launch by this command:
\app\application::open();.

1.4.Directory Structure #

The default Lemoni application structure is as follows:

# The Assets Directory
All static files related to the front-end section is located in this directory.

# The Configs Directory
Any configuration for the project like Database, email, language configuration and similar stuff is placed in this directory.

# The Controllers Directory
Contollers logic and also different version of APIs is defined in this directory.

# The Htmls Directory
All HTML files is located here.

# The Libraries Directory
Some repetitive logics which are used in controllers or models or anywhere else in the project is defined in this directory.

# The Logs Directory
This directory path is defined in index.php which is in public_html directory.
in Logs folder, all logs is stored based on time and operating system.

# The Models Directory
In this directory, all models are defined as separate files. each model has a similar name to its related database table.
